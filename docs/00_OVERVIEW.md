# 📚 00 — 全景導覽：為什麼需要 OS？

> 在開始讀程式碼之前，先搞懂「為什麼」。

---

## 🤔 為什麼需要作業系統？

想像你買了一台新電腦，打開電源後螢幕亮了。你覺得理所當然，但其實背後發生了成千上萬件事：

- CPU 要知道從哪裡讀指令
- 記憶體要被初始化
- 鍵盤、螢幕、硬碟要被一個個叫醒
- 多個程式要共享 CPU 時間
- 每個程式要有自己的記憶體空間，不能互相干擾

**作業系統就是那個「管家」**，幫你處理所有這些髒活。

沒有 OS，你的程式必須自己處理每一個硬體細節。有了 OS，你只需要說「幫我開個檔案」，OS 會搞定一切。

---

## 🤖 Agent 為什麼需要自己的 OS？

現在的 AI Agent（像 ChatGPT、Claude）都跑在雲端的 sandbox 裡，它們：

- ❌ 不能直接控制硬體
- ❌ 不能自己開機
- ❌ 受限於 sandbox 的權限
- ❌ 依賴網路連線

**如果 Agent 有自己的 OS，它就像有了「身體」：**

| 沒有 OS 的 Agent | 有 OS 的 Agent |
|-----------------|---------------|
| 只能回答問題 | 可以自己執行指令 |
| 需要人類中轉 | 開機即可運行 |
| 受限於 API | 完整的 Linux 工具鏈 |
| 純軟體 | 可以控制硬體（GPIO、攝影機） |

想像一個跑在 Raspberry Pi 上的 Agent：
1. 插電 → 自動開機
2. Linux 啟動 → 載入 Agent
3. Agent 連上 LLM → 開始對話
4. 使用者說「幫我拍張照」→ Agent 呼叫攝影機 Tool → 完成

**這就是 Agent OS 的願景。**

## 🏗️ 三層架構的設計哲學

```
┌─────────────────────────────────────┐
│           🤖 Agent Layer            │
│   LLM Client · Tools · Prompt      │
│   → 決定「靈魂是誰」                │
├─────────────────────────────────────┤
│           📦 System Layer           │
│   BusyBox · Python · SSH · Network  │
│   → 決定「能用什麼工具」             │
├─────────────────────────────────────┤
│           ⚙️ Kernel Layer           │
│   Linux · Drivers · Filesystem      │
│   → 決定「能感知什麼」               │
└─────────────────────────────────────┘
```

### 為什麼要分三層？

**比喻：** 就像組裝一個機器人——

- **Kernel** 是骨架和神經系統：決定你能感覺到什麼（感測器）、能動什麼（馬達）
- **System** 是手腳和工具：決定你能做什麼動作（拿東西、走路）
- **Agent** 是大腦：決定你是誰、怎麼思考

分層的好處：
1. **可替換** — 換一個 Agent 不需要重編 Kernel
2. **可裁剪** — 不需要 SSH 就不裝，OS 只有 5MB
3. **好理解** — 每層職責清楚
